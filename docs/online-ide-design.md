# RP2040/RP2350 Online IDE Platform Design

## 1. 项目概述
- **目标**：建设一个针对树莓派 RP2040 与 RP2350A/B 系列板卡的在线综合开发环境，兼容 MicroPython 文本开发与图形化编程流程，提供 AI 驱动的智能辅助功能。
- **用户群体**：教育机构、创客、学生与硬件开发者，覆盖从零基础到进阶用户。
- **价值主张**：一站式浏览器 IDE，免安装、跨平台，连接 USB-C 设备即可完成编程、部署、调试与学习。

## 2. 支持的硬件板卡
| 板卡 | 主要特性 | 专属适配要点 |
| --- | --- | --- |
| Stepico | RP2040 内核、外设接口丰富 | 内置串口识别、I/O 引脚模板 |
| 12 指神探 | RP2040 控制、传感器矩阵 | 传感器库、触摸按键映射 |
| 带屏的 12 指神探 | RP2040 + 屏幕 | 显示驱动、UI 预览模块 |
| RP2040 游戏机 | RP2040 + 按键/显示 | 游戏模板、音频与按键轮询支持 |
| RP2350 核心板 | RP2350A/B 可选 | 双核/AI 加速配置、内存/外设扩展支持 |

### 2.1 板卡抽象层
- 建立 **Board Profiles**：描述 USB VID/PID、串口配置、可用 GPIO、I2C/SPI/UART、屏幕参数、传感器列表。
- Profile 以 JSON/YAML 存储，可扩展，前端用于 UI 显示、后端用于代码生成与固件部署。
- 提供 **Board Capability API**：供 AI 模块与代码生成器获取板卡支持的组件与样例模板。

## 3. 核心功能模块

### 3.1 IDE 框架
- **前端**：基于 React + TypeScript 构建多标签界面，支持主题切换。
- **后端**：Node.js（NestJS）或 Python（FastAPI）REST/WebSocket 服务。
- **实时协作**（可选升级）：通过 WebRTC + CRDT（如 Yjs）实现多人同步编辑。

### 3.2 MicroPython 文本编程
- 集成 Monaco Editor，提供：
  - MicroPython 语法高亮、自动补全、Lint。
  - 与 Thonny 兼容的 REPL、文件系统浏览、脚本上传/下载。
  - 串口终端视图，支持多设备并行调试。
- **运行流程**：
  1. 浏览器调用 WebUSB/Web Serial 连接设备。
  2. 后端代理或直接通过浏览器向设备推送脚本。
  3. 返回执行日志与状态至控制台窗口。

### 3.3 图形化编程
- 采用 Blockly 引擎，实现与 Scratch 类似的积木编程体验。
- 预置板卡特定积木（GPIO、传感器、显示、音频、网络）。
- **双向同步**：
  - 积木生成 MicroPython 代码（可切换代码/积木视图）。
  - 对关键积木提供文字编辑反向同步（受限语句集）。
- **参考 UI**：
  - 左侧积木分类与搜索；
  - 中央画布；
  - 右侧 MicroPython 代码预览与运行按钮；
  - 仿 Thonny REPL 控制台。

### 3.4 AI 助手能力
1. **语法检查与对齐**：
   - 利用后端模型或 OpenAI API，对用户代码进行语法分析，给出自动修复建议，支持一键应用。
   - 自动格式化符合 Black/yapf 规范。
2. **代码优化**：
   - 对运行瓶颈、资源占用进行分析（结合静态分析和设备反馈），建议更优算法或硬件调用方式。
   - 提供“优化提示卡”，列出性能提升点与示例代码。
3. **自动注释生成**：
   - 根据函数/类行为生成多语言注释，支持一键插入。
4. **功能模块撰写**：
   - 针对 RP2040/RP2350 板卡能力（GPIO、PWM、PIO、DMA、AI 加速等）生成模块化代码骨架。
   - AI 面板可从 Board Capability API 中读取硬件资源，输出对应模板。
- **工作流**：
  - 前端通过 WebSocket 请求 AI 服务。
  - 后端维护上下文，调用推理模型，返回建议、补丁或注释。

### 3.5 设备连接与固件管理
- 使用 WebUSB/Web Serial 通过 USB-C 直连设备，兼容 Chrome/Edge。
- 提供桌面代理应用（Electron）以支持 Firefox/Safari。
- 固件管理：
  - 在线检测设备芯片型号，推荐最新固件版本。
  - 一键刷写 MicroPython 固件（UF2/DFU）。
  - 自定义启动脚本烧录与备份。

### 3.6 项目与资源管理
- 用户账户系统，支持云端保存项目、积木模板、AI 对话记录。
- Git 集成：提供一键推送到 GitHub/Gitee。
- 教学内容：
  - 课程/教程导航。
  - 示例项目库，按板卡分类。
  - 互动任务（自动评分）。

## 5. 模块设计详解

### 5.1 前端模块
- **Dashboard**：板卡选择、项目入口、教程推荐。
- **Code Studio**：
  - 代码编辑器、积木画布、终端、文件树多面板布局。
  - 可拖拽布局、全屏/分屏模式。
- **AI Assistant 面板**：聊天窗、建议列表、代码补丁 diff。
- **Device Panel**：显示连接状态、串口日志、固件版本。
- **Settings**：主题、键位、AI 模型选择、语言。

### 5.2 后端服务
| 服务 | 职责 |
| --- | --- |
| Auth Service | OAuth2/邮箱登录、Token 管理 |
| Project Service | 项目 CRUD、版本历史、云存储 |
| Device Proxy | 管理 WebUSB/Web Serial 数据流，执行脚本上传与 REPL 交互 |
| Firmware Service | 固件仓库、版本检测、刷写流程 |
| Board Service | 维护板卡 Profile、提供能力查询 API |
| AI Gateway | 调度外部 LLM 或本地模型，缓存常见任务模板 |
| Telemetry | 记录操作日志、崩溃报告，用于改进 AI 反馈 |

### 5.3 数据存储
- **PostgreSQL**：用户数据、项目元信息、AI 会话。
- **Object Storage (S3/MinIO)**：项目文件、课程资源、固件包。
- **Redis**：会话缓存、实时协作文档、速率限制。

## 6. 与参考平台对比
| 功能 | Thonny IDE | Bipes/M5Stack | 本方案 |
| --- | --- | --- | --- |
| 安装方式 | 桌面应用 | 浏览器 | 浏览器 + 可选桌面代理 |

## 7. 开发路线图
1. **MVP（第 1 阶段）**
   - 完成板卡 Profile、MicroPython 编辑/上传、基础 Blockly 积木、USB-C 连接。
   - 引入基础 AI（语法检查、注释）。
2. **阶段 2**
   - 完善图形化积木库、代码反向同步、性能优化建议。
   - 集成固件刷写、项目云存储。
3. **阶段 3**
   - 多人协作、课程系统、离线桌面代理。
   - 加入 AI 模块生成、模板市场。

## 8. 安全与合规
- 浏览器访问设备时需用户授权，采用 HTTPS + WebUSB 安全策略。
- 对 AI 数据脱敏，遵循 GDPR/中国网络安全要求。
- 提供家长/教师账户管理与学生隐私保护。

## 9. 性能与可扩展性
- 前端采用代码分包，离线缓存（PWA）。
- 后端微服务可容器化部署（Kubernetes）。
- 支持水平扩展 AI 服务，与缓存层结合减低响应延迟。

## 10. 测试策略
- 单元测试：前端组件、后端 API。
- 集成测试：设备连接模拟、AI 服务 Mock。
- 端到端测试：Cypress/Playwright 脚本覆盖文本与积木流程。
- 硬件在环测试：CI 集成真实板卡跑例程。

## 11. 未来扩展
- 支持 CircuitPython、Arduino 风格 C++。
- 引入 PIO 可视化编辑器、RP2350 推理加速示例。
- 开放插件市场，允许第三方硬件与课程接入。
